# Работа программы
Программа полностью реализует функционал прописанный в ТЗ.
Пару замечаний по пользовательскому опыту:

1. У тебя при отрисовке хода, существа сначала съедают свою "жертву", а потом делают ход в ее сторону. Я пока не смотрел как это в коде реализовано, но по первому впечатлению это немного сбивает с толку.

2. При отрисовке состояния мира в нижней строке(сущности и их общее число), у тебя сущности постоянно меняются местами. Я заглянул в код, это происходит из-за того что ты используешь словарь, а он не гарантирует сохранение порядка в общем случае. Хотя начиная с какой-то версии Питона и включили поддержку порядка, я бы не советовал от этого зависеть. Собственно тут у тебя проблема и не в этом, а в том что у тебя меняется само состояние карты, следовательно и ее "порядок сущностей".

3. Иногда проявляется баг в виде отрисовки пустого поля.
Не удалось его отловить.

```python
counts: dict[str, int] = {}
for entity in entities:
    name = type(entity).__name__
    counts[name] = counts.get(name, 0) + 1
 
   parts = [f"{self._sprites.get(name, name)} {count}" for name, count in counts.items()]
   print("  ".join(parts))
```

Лучше просто использую потом какую-то сортировку, например по именам сущностей, если тебе главное только сохранение порядка, а не его соответствие какому-то шаблону вроде - сначала неживые сущности, потом живые и т.п.
Например:

- сортировка по именам
```python
sprites = sorted(counts.keys())
parts = [f"{self._sprites.get(sprite, sprite)} {counts.get(sprite, 0)}" for sprite in sprites]
```

- сортировка по шаблону
```python
sprites = self._sort_by(counts.keys, ["Rock", "Tree", "Grass"])
parts = [f"{self._sprites.get(sprite, sprite)} {counts.get(sprite, 0)}" for sprite in sprites]

def _sort_by(self, items: List, pattern: List) -> List:
    weights = {p: i for i, p in enumerate(pattern)}
    sort_func = lambda v: weights.get(v, len(pattern))

    return sorted(items, key=sort_func)
```

## /entities и функционал сущностей
### ./entity.py
Хорошо что сделал Entity абстрактным ипользую ABC, но проблема в том, что ты все равно сможешь создать его инстанс, несмотря на то что он по логике должен быть абстрактным. Чтобы Питон выбрасывал исключения при создании класса наследуемого от ABC, в классе должна быть какая-то абстрактная сущность, которую нужно реализовать в наследниках. Имеется в виду метод с декоратором @absctractmethod или @absctractproperty.

Например:
```python
@abstractmethod
def __init__(self):
	pass
```
